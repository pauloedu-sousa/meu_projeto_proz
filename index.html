Comandos úteis para o Git



O Git é um sistema de controle de versão amplamente usado para rastrear mudanças em projetos de software e colaborar com outros desenvolvedores. Aqui estão alguns dos principais comandos do Git e como usá-los:

  1. git init: Inicia um novo repositório Git em um diretório local. ```bash git init ```


  2. git clone: Clona um repositório Git existente para o seu diretório local. ```bash git clone ```


  3. git add: Adiciona arquivos ao índice (staging area) para serem preparados para commit. ```bash git add ```


  4. git commit: Registra as mudanças no repositório com uma mensagem de commit. ```bash git commit -m "Mensagem do commit" ```


  5. git status: Exibe o status dos arquivos no diretório de trabalho e no índice. ```bash git status ```


  6. git log: Mostra o histórico de commits do repositório. ```bash git log ```


  7. git pull: Puxa as alterações do repositório remoto para o repositório local. ```bash git pull ```


  8. git push: Envia as alterações do repositório local para o repositório remoto. ```bash git push ```


  9. git branch: Lista as branches no repositório. ```bash git branch ```


  10. git checkout: Alterna para uma branch específica ou para um commit anterior. ```bash git checkout ```


  11. git merge: Mescla as mudanças de uma branch para outra. ```bash git merge ```


  12. git fetch: Busca as alterações do repositório remoto, mas não as mescla com o branch atual. ```bash git fetch ```


  13. git reset: Volta o HEAD para um commit específico. ```bash git reset ```


  14. git remote: Exibe informações sobre os repositórios remotos configurados. ```bash git remote -v ```


  15. git stash: Salva as mudanças atuais em um stash temporário. ```bash git stash ```


  16. git tag: Cria, lista ou verifica tags. ```bash git tag ```

Esses são alguns dos comandos mais comuns do Git que você pode usar para gerenciar seu repositório. Lembre-se de consultar a documentação oficial do Git e de utilizar `git --help` ou `git --help` para obter mais informações sobre cada comando específico. Além disso, pratique em projetos de exemplo para se familiarizar com esses comandos e como eles funcionam.
